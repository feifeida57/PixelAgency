## 数据流

### 1. 智能体加载流程

```
agency-agents-zh/*.md
        │
        ▼
scripts/load-agents.mjs
        │
        ▼
agent-index.json
        │
        ▼
前端导入 (import agentIndex)
        │
        ▼
Canvas 渲染
```

### 2. LLM 调用流程

```
用户发送消息
        │
        ▼
AgentLoop 接收
        │
        ▼
系统提示词 (包含 PILOTDECK.md)
        │
        ▼
LLM 决策 ──────────────────────┐
        │                       │
        ▼                       ▼
   直接回答              调用 agent 工具
        │                       │
        ▼                       ▼
   返回结果         agent({ subagent_type: "general-purpose",
                                    prompt: "你是「XX角色」..." })
                                │
                                ▼
                    SubAgentSession 执行
                                │
                                ▼
                         返回结果
```

### 3. 状态同步流程

```
agent 工具被调用
        │
        ▼
WebSocket 推送 tool_use 事件
        │
        ▼
VirtualOfficeV2 收到消息
        │
        ▼
解析 prompt 中的角色名
        │
        ▼
匹配 agent-index.json
        │
        ▼
更新智能体状态为「工作中 💻」
        │
        ▼
Canvas 重新渲染（发光动画）
        │
        ▼
tool_result 收到
        │
        ▼
恢复空闲状态
```

## 核心组件

### 1. VirtualOfficeV2.tsx

虚拟办公室主组件，负责：
- Canvas 2D 渲染
- 智能体布局计算
- 动画循环（60fps）
- WebSocket 状态同步
- 用户交互（点击查看详情）

### 3. secretary.md

LLM 秘书系统提示词，包含：
- 17 个部门列表
- 215 个智能体角色
- 调用判断标准
- 虚拟办公室状态说明

### 4. PILOTDECK.md

注入到 PilotDeck 系统提示词的文件，让 LLM 知道：
- 有 215 个可用智能体
- 如何使用 agent 工具调度
- 何时应该调用 vs 直接回答

## 技术选型

| 技术 | 选择 | 原因 |
|------|------|------|
| 渲染 | Canvas 2D | 轻量、无依赖、4GB 服务器可运行 |
| 状态管理 | React useState + useRef | 简单、够用 |
| 持久化 | localStorage | 零配置、浏览器原生 |
| 通信 | WebSocket | 实时、低延迟 |
| 智能体定义 | Markdown + YAML | 易读、易编辑、Git 友好 |

## 性能考量

- **Canvas 2D** 而非 Three.js/WebGL — 降低 GPU 依赖
- **零图片资源** — 全部程序绘制，减少 HTTP 请求
- **requestAnimationFrame** — 60fps，浏览器自动优化
- **局部重绘** — 只重绘变化的智能体，减少 Canvas 操作
- **部门分页** — 一次只渲染一个部门，避免 215 个同时渲染
